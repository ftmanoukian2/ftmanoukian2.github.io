Elementos generales de sintaxis

Indentación/contextos

En Python, una de las reglas más importantes para determinar la jerarquía o entorno de las líneas de código es la indentación (o separación horizontal).
La indentación se realiza utilizando la tecla TAB, o en su defecto cuatro espacios en lugar de un TAB.
La primer línea de cada archivo de código debe comenzar sin separación horizontal. Es decir, el texto correspondiente a la primer acción debe estar alineado con la columna 0 del archivo de código.
La separación horizontal delimita la extensión del código correspondiente a los siguientes contextos:

  • Condicionales
  • Bucles
  • Funciones
  • Clases
Para iniciar un contexto, en la línea que da inicio al mismo se utiliza ":", tras lo cual el lenguaje esperará la separación horizontal.
Al "romper" la separación horizontal, el contexto de la acción o línea de código vuelve a ser el mismo que el del elemento que dió inicio a la separación. Por ejemplo, una línea de código con la misma separación que un condicional se ejecuta independientemente del resultado de la evaluación.

Ejemplos
Condicional 
                
if condicion == True:
    accion1 # Esta acción se realiza sólo si la condición se cumple

accion2 # Esta acción se realiza independientemente del resultado de la evaluación
Bucles 
                
while variable < 10:
    variable = variable + 1 # Se realiza dentro del bucle

variable = 0 # Se realiza una vez, al finalizar el bucle
Funciones 
                
def nombre_de_funcion():
    accion1 # acción realizada al llamar a la función
    return valor_de_retorno # valor devuelto por la función

accion2 # acción NO realizada por la función 
# generalmente sólo se utiliza esta separación para definir otras funciones/clases, 
# no para acciones a realizar secuencialmente por el programa.

Comentarios

Los comentarios son elementos de texto dentro del código fuente de un programa que no son ejecutados por el intérprete. Sirven para proporcionar explicaciones, anotaciones o documentación sobre el código.
Existen en dos formas: de una línea o de más de una línea. En ambos casos, su indentación es libre y no sigue las reglas necesarias en el resto del código. 

        
# Esto es un comentario de una línea. 

# Puede ser escrito tanto en una línea vacía,

print("hola!") # como también a continuación de una sentencia de código

# Cada línea de comentario debe comenzar con un '#', 
# de lo contrario se interpreta como código

Los comentarios de una línea son especialmente útiles para explicar el funcionamiento de líneas de código particulares, aunque se vuelven tediosos cuando se desea escribir un texto más largo. 

        
"""
Esto es un comentario multilínea. Entre los delimitadores (tres comillas dobles) 
se puede escribir una cantidad indeterminada de líneas de comentario, 
sin necesidad de utilizar delimitadores en cada una.
"""

Los comentarios multilínea son más apropiados para realizar explicaciones conceptuales acerca del programa, pero también para describir tipos de argumentos y valor/es de retorno.
También son especialmente útiles para brindar la información en pantalla que el IDE muestra cuando detecta el llamado a una función o módulo. Para esto, deben ser incluidos al inicio de la función o módulo.
