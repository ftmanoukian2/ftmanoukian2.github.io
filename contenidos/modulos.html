Módulos

Los módulos son archivos que contienen funcionalidad (módulos/clases) pero no constituyen un programa en sí. Son especialmente útiles para almacenar funcionalidad no específica del programa, ya que permiten una fácil reutilización del código, aunque también son una buena forma de separar la implementación del programa principal.
Son la forma estándar de compartir código, por lo que toda la funcionalidad externa la encontraremos en forma de módulos.

Forma de creación

Los módulos son archivos con extensión .py.
Si el módulo es de creación propia, es conveniente que se encuentre en la misma ruta (o carpeta) que el archivo desde el que se lo importa.

Sintaxis

  • Dentro del módulo

    Para el intérprete, un módulo es indistinguible de un archivo ejecutable ya que comparten tanto sintaxis como extensión. Las funciones o clases que puedan estar contenidas en un módulo deben respetar las reglas de indentación.

  • Importación

    El lenguaje es muy flexible a la hora de importar módulos, y existen varias formas de hacerlo.

    • Módulo completo con referencia

      Esta forma de importar requiere que cada llamado a una función contenida en el mismo sea referenciada.
      Esto permite identificar fácilmente el origen de una función cuando el programa se vuelve largo o complejo, y es absolutamente necesario cuando existen funciones con el mismo nombre en diferentes módulos.
      La desventaja es que los llamados a función se vuelven largos, con lo cual el programa se vuelve más tedioso de leer. Para esto, existe la siguiente alternativa. 

                              
# se importa el módulo utilizando el nombre del archivo
# sin la extensión
import nombre_de_modulo

# para llamar una función dentro del módulo, se debe referenciar
# al mismo utilizando su nombre
nombre_de_modulo.nombre_de_funcion_en_modulo()

    • Módulo completo con referencia y alias

      Similar al anterior, permite abreviar el nombre del módulo, contribuyendo a una mayor facilidad de lectura del programa. 

                              
# se importa el módulo pero se especifica un "alias"
# más corto que el nombre para referenciar al mismo
import nombre_de_modulo as ndm

# para llamar una función dentro del módulo, se debe referenciar
# al alias
ndm.nombre_de_funcion_en_modulo()

    • Funciones específicas sin referencia

      Esta forma permite importar funciones o clases sin utilizar la referencia al módulo.
      En proyectos cortos, o con pocos módulos, permite más facilidad a la hora de escribir el programa aunque no es recomendado en proyectos más grandes. 

                              
# en lugar de importar el módulo completo, se especifica su nombre
# y la función (o funciones, separadas con coma) a importar
from nombre_de_modulo import nombre_de_funcion_en_modulo

# la función importada de esta forma no debe ser referenciada
# al módulo que la contieen
nombre_de_funcion_en_modulo()

    • Otras formas

      Las últimas dos formas se pueden combinar, permitiendo utilizar un alias para llamar a una función individual sin referenciarla. Esto permite reducir aún más el largo del programa, aunque dificulta la comprensión del mismo a medida que crece.
